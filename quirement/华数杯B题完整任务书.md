# 华数杯B题网络切片无线资源管理方案设计

## 📋 问题背景

随着物联网设备的指数级增长，移动通信需求呈现爆发式增长，推动了网络架构的不断演进。为应对不同场景下的数据密度与服务需求，移动通信网络逐步采用宏基站与小基站混合部署的异构蜂窝网络结构，其中宏基站负责大范围的覆盖，微基站用于在高密度区域提供高效的无线接入。两层的异构网络可以提升网络容量、覆盖范围和用户体验，尤其是在5G及未来网络中，以支持大规模的设备接入和高数据传输速率。

## 🏗️ 网络架构描述

### 异构网络结构
**宏基站**：负责大范围网络覆盖
**微基站**：高密度区域高效无线接入
**两层网络**：提升容量、覆盖范围、用户体验

### 网络切片技术
通过网络功能虚拟化(NFV)将物理网络划分为多个逻辑独立的网络"切片"：

#### 三类核心切片：
- **URLLC切片**（Ultra-Reliable and Low-Latency Communications）
  - 场景：工业控制、医疗、自动驾驶
  - 特点：高可靠性、低时延
  
- **eMBB切片**（Enhanced Mobile Broadband）
  - 场景：超高清视频、虚拟现实、云游戏
  - 特点：大带宽、高下载速率
  
- **mMTC切片**（Massive Machine Type Communications）
  - 场景：智能家居、智能城市、工业自动化
  - 特点：大规模设备连接、低功耗、低成本连接

## 🔄 技术原理

### 资源块定义
- **频域宽度**：360kHz
- **时域宽度**：10ms
- **正交频分多址(OFDMA)**：将频谱划分为正交子信道
- **最小分配单位**：资源块(RB)
- **同频复用**：微基站之间可重复使用频率（引入干扰问题）

## 🎯 评估标准

### 服务质量评估模型（附录3完整定义）

#### **URLLC用户服务质量**
QoS计算遵循严格数学公式：
```
y^URLLC = α^L  (当 L ≤ L_SLA 时),
          = -M (当 L > L_SLA 时)
```
- α：效用折扣系数（0.95）
- L：实际任务延迟（传输+排队）
- L_SLA：最大允许延迟（5ms）
- M：惩罚系数（5）

#### **eMBB用户服务质量**
```
                  { 1.0              (当 R ≥ R_SLA 且 L ≤ L_SLA)
y^eMBB = min(R/R_SLA, 1.0) (当 R < R_SLA 且 L ≤ L_SLA)
                  { -M               (当 L > L_SLA)
```
- R：实际传输速率
- R_SLA：速率SLA阈值

#### **mMTC用户服务质量**
```
             { Σc'i/Σci  (当所有用户延迟 ≤ L_SLA)
             { -M        (当有用户延迟 > L_SLA)
```
- c'i：成功接入用户数（延迟达标）
- ci：有任务的总用户数

### 核心性能指标
- **延迟计算**：L = 排队延迟 + 传输延迟
- **速率计算**：遵循香农公式
- **信道模型**：附录1信号传输模型
- **干扰机制**：附录5信号干扰模型
